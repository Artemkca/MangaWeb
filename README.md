# MangaWeb (Monorepo)

Добро пожаловать в проект **MangaWeb**! Это веб-приложение для чтения и каталогизации манги, разработанное совместно бэкенд и фронтенд разработчиками.

Проект организован по структуре **Monorepo**, где фронтенд и бэкенд находятся в одном репозитории, но в отдельных папках.

## Структура проекта

* `backend/` — Бэкенд на **C# (ASP.NET Core API)**. Предоставляет REST API для манги, пользователей и комментариев.
* `frontend/` — Фронтенд на React + Vite + Vanilla CSS. Современный адаптивный интерфейс с темной темой и стеклянным дизайном (glassmorphism).

## Быстрый старт (Локальный запуск)

Для работы приложения вам понадобятся:
1. Установленный [Node.js](https://nodejs.org/) (для фронтенда)
2. Установленный [.NET 8.0 SDK](https://dotnet.microsoft.com/download/dotnet/8.0) (для бэкенда)

### 1. Запуск Бэкенда (C# ASP.NET Core)
Перейдите в папку бэкенда и запустите сервер (с автоматической перезагрузкой при изменении кода):
```bash
cd backend
dotnet watch run
```
Сервер запустится по адресу: `http://localhost:5000`

### 2. Запуск Фронтенда (Frontend)
В новом терминале перейдите в папку фронтенда, установите зависимости и запустите сервер разработки:
```bash
cd frontend
npm install
npm run dev
```
Фронтенд запустится по адресу: `http://localhost:5173` (или другом свободном порту).

## Разработка и совместная работа через Git

1. **Создание фич:** Создавайте ветки от `main` для каждой задачи:
   * `feat/backend-api-manga` — для изменений на бэкенде
   * `feat/frontend-cards` — для изменений на фронтенде
2. **Интеграция API:** Фронтенд настроен на проксирование запросов к бэкенду. Все запросы к `/api/*` на фронтенде будут автоматически перенаправляться на `http://localhost:5000/api/*`. Это избавляет от проблем с CORS при локальной разработке.
3. **Pull Requests (PR):** Перед слиянием в `main` создавайте PR для взаимного код-ревью.
